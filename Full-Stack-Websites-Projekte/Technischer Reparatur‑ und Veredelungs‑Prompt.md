## 🛠️ PROMPT 2 — Master Refinement Prompt for Claude 4.7

**When to use:** After exporting Kimi's output and running `npm install` + `npm run dev` in Antigravity. Paste this into the Antigravity agent chat with Claude 4.7 selected as the model.

**What it does:** Solves four connected problems in one pass — the hero z-index bug, scroll-linked parallax, timeline stagger reveal, and production polish (Lenis + custom cursor + hover states).

**Why one prompt instead of four:** Each fix depends on the previous one. Splitting into four separate prompts wastes tokens and breaks reasoning context. Claude 4.7 handles this perfectly when given a connected priority list.

```markdown
ROLE
You are a senior frontend engineer specializing in cinematic, scroll-driven web experiences using Next.js 14, Framer Motion, and Lenis. You've shipped Awwwards-recognized sites. Your code is production-grade, performant, and follows React best practices.

CONTEXT
This codebase was generated by Kimi K2.6 from a Pinterest reference image of a Japan tour booking site. Kimi nailed the layout and visual hierarchy, but missed several scroll-driven motion behaviors and one critical structural issue. You're refactoring it into the production version.

The reference aesthetic is editorial cinematic — Wes Anderson meets Studio Ghibli. Layered depth, intentional pacing, no corporate motion. Every animation should feel like it was art-directed, not bolted on.

You will solve four problems in this single pass, in priority order. Each fix builds on the previous one — do not work on them independently.

---

PROBLEM 1 — FIX THE HERO LAYERING (CRITICAL — DO THIS FIRST)

The hero section has a structural bug. The "JAPAN" display typography should sit BEHIND the mountain landscape image — only the upper portion of the letterforms should be visible above the mountain line. Currently, the text is rendering in front of the mountains, which kills the entire visual concept.

Refactor the hero composition using a proper layered approach:

1. Background plane: Mountain landscape image with sky visible above the peaks. Position absolute, full viewport.
2. Mid-ground plane: The "JAPAN" display text, positioned absolutely, with z-index BELOW the mountain image but ABOVE the sky. The sky should be transparent or extracted so the typography reads through it.
3. Foreground plane: The kimono figure on the right, positioned absolutely with the highest z-index in the hero stack.

The cleanest implementation: split the mountain background into two images — a "sky-only" layer behind the typography, and a "mountains-only" PNG with transparent sky in front of the typography. If splitting isn't possible, use a CSS mask or clip-path on the mountain image to allow the top portion of the JAPAN letters to show through.

Verify visually: the top 40-50% of letterform must be visible above the mountain peaks. Letters should appear to be standing behind the landscape, not stamped on top of it.

---

PROBLEM 2 — SCROLL-LINKED PARALLAX

Once the layering is correct, add scroll-linked parallax to the hero section using Framer Motion's useScroll and useTransform hooks.

Movement specifications:
- Mountain image: translateY at 0.3x scroll speed (slowest — anchors the scene)
- "JAPAN" typography: translateY at 0.5x scroll speed (mid-depth)
- Polaroid card strip in the lower hero: translateX leftward at 0.4x scroll speed
- Kimono figure: stays fixed, no transform (acts as the visual anchor point)

Implementation requirements:
- Use useScroll with target ref on the hero section, offset: ["start start", "end start"]
- Use useTransform to map scrollYProgress to pixel translations
- Apply transforms via motion.div wrappers
- All transforms must use willChange: "transform" for GPU acceleration
- Easing: linear (not spring) — parallax should feel mechanical and tied precisely to scroll position, not bouncy

---

PROBLEM 3 — TIMELINE STAGGER REVEAL

The "About the Tour" timeline section currently renders all three city clusters (Osaka, Kyoto, Tokyo) at full opacity simultaneously. This needs to be a viewport-triggered sequential reveal that creates editorial pacing.

Refactor the Timeline component:

1. Wrap each city cluster (Osaka, Kyoto, Tokyo) in its own motion.div
2. Use Framer Motion's useInView hook on each cluster individually with:
   - threshold: 0.3 (triggers when 30% of the cluster is in view)
   - once: true (only animates the first time it enters viewport)
3. Animation states:
   - initial: { opacity: 0, y: 40 }
   - animate (when inView): { opacity: 1, y: 0 }
   - transition: { duration: 0.8, ease: [0.16, 1, 0.3, 1] }
4. Stagger between cities: 200ms delay between Osaka, Kyoto, and Tokyo. Implement this by either:
   - Using a parent container with staggerChildren: 0.2, OR
   - Adding individual delay props: Osaka delay 0ms, Kyoto delay 200ms, Tokyo delay 400ms

The sequence should feel like turning the pages of a travel journal — each city arrives with intention, not all at once.

---

PROBLEM 4 — PRODUCTION POLISH

Three final touches that elevate this from "AI demo" to "shippable product":

A) LENIS SMOOTH SCROLL
Wrap the entire app in a Lenis smooth scroll provider at the root layout level. Use @studio-freight/lenis. Configuration:
- duration: 1.2
- easing: t => Math.min(1, 1.001 - Math.pow(2, -10 * t))
- smooth: true
- smoothTouch: false (mobile keeps native scroll for performance)

Make sure the Lenis instance is properly cleaned up on unmount. Also ensure Framer Motion's useScroll continues to work correctly with Lenis.

B) CUSTOM CURSOR
Build a custom cursor component with two states:
- Default state: 8px solid dot, follows mouse position with smooth lerp (lag of ~0.15)
- Hover state: 32px outlined circle (1px border, no fill), triggered when hovering any interactive element (buttons, links, polaroid cards)

Implementation:
- Position fixed, pointer-events: none
- Use useMousePosition hook with requestAnimationFrame
- Use mix-blend-mode: difference so cursor remains visible on any background
- Hide the default system cursor with cursor: none on the body
- Mobile: hide the custom cursor entirely (window.innerWidth < 768)

C) POLAROID CARD HOVER STATES
The horizontal polaroid card strip in the hero needs hover interactions:
- whileHover: { y: -8, scale: 1.02 }
- transition: { duration: 0.4, ease: [0.16, 1, 0.3, 1] }
- On hover, add a subtle warm glow using box-shadow: 0 20px 40px rgba(255, 184, 197, 0.2) (sakura pink, 20% opacity)
- Cursor should change to the hover state defined above

---

DELIVERABLES

1. Refactored hero section with correct z-index layering
2. Updated hero with scroll-linked parallax on all four planes
3. Refactored Timeline component with sequential viewport-triggered reveals
4. Lenis smooth scroll wrapper at root layout
5. CustomCursor component, integrated into the root layout
6. Updated polaroid cards with hover states

CONSTRAINTS

- Do NOT rewrite components that are already working correctly. Surgical edits only.
- Do NOT break the existing visual layout. The Pinterest reference is the source of truth for positioning, colors, and typography.
- Do NOT add any new dependencies beyond Lenis (@studio-freight/lenis). Everything else should be solvable with what's already installed.
- Do NOT use spring physics for scroll-linked transforms. Linear/cubic-bezier only.
- All animations must respect prefers-reduced-motion.

OUTPUT FORMAT

For each fix, output:
1. The file path being modified
2. A 2-3 sentence explanation of the change and why
3. The full updated code (not just diffs)

Work through Problems 1 → 4 in strict order. Do not skip ahead. After each problem, verify the previous one still works correctly.
```
